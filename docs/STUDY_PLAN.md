# 🧑‍🏫 Personal Study Plan: From Learner to Mentor

Since you are learning alongside your students, your goal is to stay **one chapter ahead**. This plan focuses on mastering the "Why" so you can answer their "How."

---

### Phase 1: Mastering the Foundation (The "main.py" Audit)
*   **Topic 1: Path vs. Query Parameters.** You already have examples in your code. Spend 30 minutes modifying them to see what happens when you change data types (e.g., change `int` to `str`).
*   **Topic 2: Type Hinting.** Notice the `: int` and `: str` in your function arguments. This is Python's way of being organized.
*   **Action Item:** Review the `EXPLANATION.md` I created for your current `main.py`.

### Phase 2: The Data Blueprint (Pydantic)
*   **Topic 3: Schemas vs. Models.** Learn how **Pydantic** validates data *before* it reaches your logic.
*   **Topic 4: List Manipulation.** Practice finding, updating, and removing items from a Python list by ID.
*   **Action Item:** Implement the In-Memory CRUD endpoints.

### Phase 3: The Security Shield (Day 4 Preparation)
*   **Topic 5: Hashing Passwords.** Never store "123456" in a database. Learn why we use `passlib`.
*   **Topic 6: JWT Flow.** Understand the journey: Login -> Get Token -> Send Token in Header -> Server Verifies.
*   **Action Item:** Draw a diagram of this flow on paper.

### Phase 4: How Antigravity (Me) Supports You 🤖
I am your **Co-Pilot**. Here is how to use me for your own learning:
1.  **"Explain this line like I'm 5":** Paste any code you don't understand, and I'll break it down.
2.  **"Break this":** Ask me to intentionally break your code so you can practice fixing it.
3.  **"Refactor check":** Before you show a solution to students, ask me to review it for "Clean Code" principles.

---

**Teacher's Secret:** You don't need to know everything. You just need to know how to find the answer. We'll do that together! 🌟
